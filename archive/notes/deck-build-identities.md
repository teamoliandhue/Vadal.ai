The landing page (`src/app/page.tsx` → renders `src/app/journey.tsx`, a `"use client"` component) is a pitch deck (user explicitly asked for "pitch deck style", not a website, 2026-06-10): full-screen `h-screen snap-start` slides inside an inner `snap-y snap-mandatory` scroll container (`deckRef`), with fixed deck chrome — header (wordmark + review date), footer counter + slide label + prev/next arrows + accent progress bar, right-side dot rail with hover tooltips. Current slide tracked by a scroll listener (`round(scrollTop/clientHeight)`); `go(i)` smooth-scrolls. A verdict slide is inverted dark — chrome colors flip via a `dark = slide === N` check with 500ms transitions.

**Design language = gusta.studio-inspired** (user rejected an earlier "boring and cluttered" version on 2026-06-10). Warm paper canvas, oversized display type, editorial serif-italic accent words, marquee, big premium cards, one calm indigo accent, an inverted near-black Verdict section. Implemented via:
- **Fonts** in `layout.tsx`: Inter (body) + **Space Grotesk** (`--font-grotesk`, display) + **Instrument Serif** (`--font-serif`, italic accent words). All three variables on `<html>`.
- **`.studio-root` scope** in globals.css (end of file) with `--st-*` tokens (canvas `#f1eee7`, ink `#18171c`, accent `#5266eb`). Helpers: `.st-display` (Space Grotesk, tight), `.st-serif-it` (italic accent), `.st-eyebrow`, `.st-card`, `.st-pill`/`.st-pill-ink`/`.st-pill-ghost`, `.st-link` (underline grow), `.st-marquee`+`.st-marquee-track` (keyframe `st-marquee`), `.st-invert` (dark section — overrides `--st-*` to cream-on-near-black).

**Branding (final deck state, 6 slides):** the "Branding Identities" slide shows SEALED CARDS only (no logos — client explicitly wanted marks hidden on the deck) linking to `/branding/<id>` brand case-study pages. **Real logo assets** (client-supplied SVGs) lived in repo-root `branding/` → copied to `public/branding/{logo-1..5,icon-1,icon-2}.svg`; mark gradient #8DA2FF→#5D63E1, accent red #FB4B43. **Case-study pages** (`src/app/branding/[id]/page.tsx` server wrapper awaiting `params` Promise + `study.tsx` client): dark editorial layout — hero lockup img, "Why this symbol" (anatomy table), "How it connects to Vadal" (feature mapping), "Why these colours", and a **colour-experiment section**: 5 palettes re-tint the mark live via `src/app/branding/marks.tsx` (VadalIcon1–5 — exact client vectors with gradient stops + accent as props). **Five identities:** 01 "Signal" (V-flag w/ lens aperture + taper + spark), 02 "Orbit" (circle + pulse-check line + open inlet + spark), 03 "Weave" (woven translucent strands), 04 "Compass" (four arcs forming a negative-space spark), 05 "Huddle" (three person-badges). VadalIcon3–5 were SCRIPT-GENERATED from the SVGs (python: parameterize gradient ids → `${id}-pN`, map hex stops → from/to/accent + `tint()`-derived light slots). GOTCHA from that generation: `fill="url(#...)"` refs need their own regex pass — missed ones render invisible paths.

**Mobile (June 11):** the deck's snap-scroll is DESKTOP-ONLY — on small screens snap-mandatory + h-screen + overflow-hidden clipped tall slides. Pattern: `Slide` = `min-h-screen overflow-visible lg:h-screen lg:snap-start lg:overflow-hidden`; deck container = `overflow-y-auto lg:snap-y lg:snap-mandatory`; fixed header gets `bg-[#f1eee7]/85 backdrop-blur-md lg:bg-transparent`; footer counter + prev/next arrows are `hidden lg:flex` (mobile keeps only the progress bar).

**Preview gotchas:** (1) the Claude_Preview screenshot tool only paints the document-top viewport band — *window* scrolling then screenshotting gives blank frames; but scrolling an **inner container** (like the deck's snap container) works fine, so navigate slides via `document.querySelector('.snap-y').scrollTo(...)`. (2) Screenshots often catch CSS transitions/React re-renders mid-flight — take a second screenshot before concluding something is broken.
