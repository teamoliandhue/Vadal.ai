# Build notes — the deck's 7 product-design style variants (HISTORICAL)

> Archived from the `vadal-ai-style-variants` memory so it no longer loads into every session.
> Read ONLY if working on the frozen deck (`apps/deck`) or recounting the journey. NOT needed
> for the product build. Paths are pre-reorg: `vadal-dashboards/src` → `apps/deck/src`.

Each design direction is a **self-contained route** `src/app/<slug>/page.tsx` with all its section components inline. Shared layer: `@/lib/data.ts` (all numbers), `@/components/charts.tsx` (ScoreRing, ArcGauge, Sparkline, TrendChart — they theme off CSS vars `--ink`/`--faint`/`--bench`/`--card`/`--gauge-track` and accept `color` props). A variant goes live by flipping its `styleDeck` entry `status: "soon" → "ready"` in data.ts (the landing page auto-links ready ones to `/<slug>`).

**All 5 built and "ready":** `lumen` (editorial), `glass` (iOS-26 liquid glass), `minimal` (`--mn-*` tokens, monochrome type-led), `fintech` (`--ft-*` tokens, Mercury-style: warm-gray canvas, floating white cards w/ soft shadow, periwinkle #5266eb, superscript numbers via SupNum, soft tinted chips, quick-action pill row), `genalpha` (`--ga-*` tokens, **rebuilt 100% on Cal AI (mobbin)** after two earlier skins — Duolingo ("kindergarten", rejected) then Withings+gradient; user finally asked for a total Cal AI rebuild renamed from "Playful" to **"GenAlpha"** (route `/genalpha`, playful folder deleted, styleDeck slug replaced). Cal AI signatures: flat light-gray canvas #f2f2f4 with radial white sheen at top, borderless white cards radius 22 (shadow-only), **BLACK as the brand color** (`--ga-brand` flips black↔white in dark mode; black hero ring, black meters, black active-nav + period pills, black FAB bottom-right `.ga-fab`, black `.ga-btn` CTA), huge 800-weight numbers (hero 56px), thin `Ring` gauges w/ rounded caps + tiny icon centers, small accent pops only (red/orange/blue/yellow/green/purple ~#e8584c etc.), **week strip** (`.ga-day`: dashed rings = past check-ins, solid black = today; ties to streak), **yellow health-score bar** + narrative inside hero, 🔥/⚡ white pillboxes (`.ga-pillbox`), gray `.ga-chip` w/ colored dots, missions w/ dashed-circle numbers. Gamified mechanics retained: missions+XP, streak, team league w/ zones, medals, mood emoji. **"Cool layer" on top:** film-grain overlay (`.genalpha-root::after`, fixed, SVG fractalNoise, opacity .035/.05 dark), CountUp client component (rAF count-up in genalpha-chrome.tsx — all setState inside rAF callbacks because Next 16 `react-hooks/set-state-in-effect` rejects sync setState in effects), live signal ticker marquee (`.ga-ticker`, doubled items, edge mask-fade, pause on hover), flame-flicker + ping-dot keyframes, springy ring scale on card hover, FAB + rotates on hover, staggered `rise rise-1..4` row entrances; all gated by prefers-reduced-motion. **JSX gotcha hit here: a space between `</b>` and following text on the same line got collapsed at render in Next 16 — use explicit `{" "}` after inline elements in prose.** All theme light+dark via the shared `dark` class + `vadal-theme` localStorage key; each route has its own `<slug>-chrome.tsx` toggle using the `useSyncExternalStore` pattern. **Card-balance technique used everywhere:** list rows get `flex flex-1` inside a `flex flex-col` card so content distributes to fill the stretched grid height (verified via DOM measurement — trailing gap ≈ padding only).

**6th variant `volt` (June 2026):** built from the client's Figma (file jVJrJdywBIeOZZfL8yXwG3, node 291:12) — "mission-control" Executive Dashboard. `--vt-*` tokens; **dark mode IS the reference design** (canvas #080809, gradient charcoal cards #1a191c→#0f0f11, volt-lime #c6f82a accent, red #ff452e risk, hatched orange voices bar via repeating-linear-gradient, lime left-border active nav, green/red radial glow heroes `.vt-glow-green/.vt-glow-red`); lite mode is a derived light adaptation (accent fill #b8ea18, accent text #5d7a06 for contrast). Space Grotesk (`.vt-num`) stands in for Figma's Circular Std. NOTE: the landing deck does NOT read styleDeck from data.ts — it has its own hard-coded `STYLES` array in `src/app/journey.tsx` (~line 208) that must be updated too.

**7th variant `ticker` (June 2026):** from the client's Figma (same file, node 291:519, frame misleadingly named "Dark Neon (Gen Z)" — it's actually light). Markets-desk aesthetic: cool paper-gray canvas #f0eff2, FLAT borderless cards #e9e8eb (radius 24, no border/shadow — cards barely darker than canvas), ticker-yellow #f5c518, green #60d399 / red #ff452e, centered top pill-nav (black active pill w/ yellow square dot), full-width "tape" chart (1px micro-bars + amber engagement line + gray participation line + translucent amber brushed range + scrubber + 4AM–4PM axis), segmented half-arc mood gauge, AI-theme slider rows w/ black ▲ markers, dot-matrix manager heatmap (DotBlock svg grids), waffle recognition chart. `--tk-*` tokens; **light mode IS the reference**, dark is derived (canvas #131312, nav pill inverts via --tk-pill-bg/fg). Colors were sampled from the Figma screenshot PNG with PIL. Gotcha: React Compiler lint `Cannot reassign variable after render completes` rejects `let offset` accumulation in component render — use reduce. Preview tool gotchas: `preview_resize` preset "desktop" can collapse the viewport to 1px wide (always pass explicit width/height); screenshots taken while window-scrolled composite incorrectly (blank band) — resize taller and scroll to 0 instead.

**Glass specifics:** styled after **iOS 26 Liquid Glass** (user supplied reference images): deep navy/indigo mesh with electric-blue + violet glows, smoky dark glass with luminous top rim + inner glow, periwinkle-lavender CTA pills, blue/periwinkle accents (no teal/pink rainbow — pink only survives in the vadal brand mark and semantic reds). **User-selectable background presets** (GlassBgPicker in glass-chrome.tsx, palette button in TopBar): per-mode choices via `data-glbg-light` / `data-glbg-dark` attrs on `<html>` + localStorage (`vadal-glass-bg-light/dark`); CSS preset blocks in globals.css keyed `html.dark[data-glbg-dark="..."]` / `html:not(.dark)[data-glbg-light="..."]`. Dark: ocean (default, = reference image), violet, aurora, graphite. Light: sky (default), lilac, mint, dawn. Glass-native layout (floating slim icon rail + detached frosted top bar + asymmetric bento) over an animated gradient mesh. All glass CSS is scoped under `.glass-root` in globals.css (`.glass-card`, `.glass-soft`, `.glass-mesh`/`.glass-mesh-blobs`, `.glass-cta`, `--gl-*` tokens that flip under `.dark .glass-root`). Theme toggle in `glass-chrome.tsx` uses `useSyncExternalStore` (reads the `dark` class) — NOT `useState`-in-effect, which the Next 16 lint rule `react-hooks/set-state-in-effect` rejects (Lumen's older `theme-toggle.tsx` still uses the rejected pattern).

**GOTCHA (cost a debug cycle):** a custom component class like `.glass-card` that sets `position` will OVERRIDE Tailwind's `fixed`/`absolute`/`relative` utilities — same specificity (0,1,0), but globals.css comes later in source order than the Tailwind layer, so it wins. Symptom: a `fixed` rail rendered in-flow and pushed all content down. Fix: never put `position` in these primitives; leave positioning to utility classes.
