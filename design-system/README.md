# Design system

Home for the Vadal design-system **documentation** — color, type, the logo/spark rules,
spacing, motion, and component usage.

Current plan: the *implemented* component library and tokens live in code inside
`apps/product`, with a `/design-system` documentation route there so docs stay in sync with
the components. This folder is the reserved home if we later want standalone docs or extract
the system into a shared package consumed by more than one app.

Empty for now — populated during Stage 2.
